The Core Premise

This isn't project management.

It's narrative continuity — making sure every session starts where the last one ended, not from zero.

The contract defines what OpenClaw tracks, what it surfaces, and what it keeps quiet. The goal: coaching that compounds, not resets.

Two Parties, Different Needs

Founder
Jai & Satith

Needs: accountability, learning prompts, no micromanagement

Mentor
Craig

Needs: pre-session briefs, pattern alerts, drift signals

Visibility Matrix

What's surfaced vs. kept quiet

Data Type To Founder To Mentor
Commitment status Yes — prompts + updates Yes — in briefs

Hypothesis evidence Yes — weekly prompt Yes — in briefs
Implicit commitments Soft — bi-weekly nudge Yes — in briefs
Enthusiasm signals No Yes — pattern alerts

Rule: Founders see tasks and prompts. Mentors see patterns and signals.

4/8
Escalation Rules

When OpenClaw gets louder

Default mode is quiet. Escalation happens only when patterns emerge:

Level 1
Soft prompt to founder (deadline approaching, hypothesis check)
Level 2
Prompt to both parties (deadline missed, no response)
Level 3
Alert to mentor only (pattern detected: 3+ occurrences)
Level 4
Flag for human review (founder disengaged, critical blocker)

Human in loop: Level 4 always requires Craig's review before any action.

5/8
Trust Boundaries

What OpenClaw never does

  • Never scolds founders for missed deadlines
  • Never shares founder's emotional signals with founder
  • Never makes coaching decisions — only surfaces data
  • Never contacts external parties (Marc Andreessen, etc.)
  • Never auto-generates emails or messages without approval

Founder Privacy Protections

  • Emotional signals (enthusiasm, stress) are mentor-only
  • Implicit commitments never trigger escalation
  • Founder can mark any commitment as "private" — excluded from mentor view

Design principle: OpenClaw is a memory aid, not a manager. It surfaces what humans would forget, without replacing human judgment.

6/8
The Contract Terms

What each party agrees to

Founder Agrees
  • Respond to status prompts (even "blocked")
  • Answer hypothesis checks with evidence, not tasks
  • Flag blockers early, not at deadline
  • Trust that emotional signals are private
Mentor Agrees
  • Read pre-session briefs before meetings
  • Use pattern alerts as coaching input, not judgment
  • Respect founder privacy boundaries
  • Review Level 4 escalations within 48hrs

OpenClaw Agrees: Hold the narrative steady. Surface what matters. Stay quiet when appropriate. Never replace human judgment.

7/8
The Outcome

What this enables

Without this contract:

  • Sessions start with "Where were we?"
  • Commitments drift, then disappear
  • Patterns go unnoticed until it's too late
  • Coaching resets instead of compounds

With this contract:

  • Sessions start with "Last time you committed to X — what happened?"
  • Commitments have status, visibility, accountability
  • Patterns surface early: "Jai's said 'not excited' for 3 sessions"
  • Coaching compounds: each session builds on the last

"This isn't about managing founders. It's about holding the narrative steady so Craig's coaching compounds instead of resets."
