1/5

Where I Started

I thought I'd install the browser extension on 10-20 attorneys and watch what happened.

That was the test Neal pushed: get it in real browsers, iterate on feedback, prove the distribution hack works.

But we hit a hard blocker:

Can't distribute the extension safely without either exposing our entire codebase OR making attorneys enable developer mode and load from a zip file.

That's too much friction. Attorneys won't do it. Firms probably won't allow it.

2/5

What I Discovered

Pivot #1: Demo Instead of Install

If I can't distribute, I can still validate. Demoed to 4 lawyers so far.

✓ All loved it

✓ Wanted more detailed reports (liked the concept)

✓ Wanted Cloak immediately (validates both products)

✓ 2 solo practitioners: wrong fit (no staff), but said "this would slot in nicely" at mid-size firms

✓ 2 firm attorneys: passing our info to managing partners (buying decision makers)

The Tension That Surfaced

Attorneys want manager dashboards to check employee usage.

Our architecture: everything runs locally. Data never leaves the device.

Trade-off: Do we sync metadata (risk scores, counts) to the cloud while keeping prompts local? Or stay pure local and lose organizational visibility?

3/5

The Real Opportunity

While solving the distribution problem, we built the entire backend infrastructure for a real SaaS launch:

  • ✓ NER model tested, ready to plug in
  • ✓ Auth system: Supabase + Resend OTP
  • ✓ Payment processing: Stripe webhooks + edge functions
  • ✓ Pricing locked: $7.99/month, $69.99/year
  • ✓ Ticketing system for bugs/feedback
  • ✓ Complete onboarding flow
  • ✓ Chrome Web Store account: verified, paid, ready
  • ✓ Microsoft Edge Add-on Store: verified, paid, ready

The distribution blocker? Solved.

We just need to upload the extension for review to Chrome and Edge stores.
