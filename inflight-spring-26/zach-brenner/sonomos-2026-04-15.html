1/5

Where I Started

Desktop-first, network interception

The plan was clear: build a desktop app that intercepts network traffic to AI providers, modify the content locally, and send it on its way.

  • Desktop as primary product
  • Network interception as the mechanism
  • Extension as fallback or secondary
  • Focus on IT/enterprise credibility

The Assumption

If we could read the traffic, we could modify it. Desktop would give us that control.

2/5

What I Discovered

Interception works. Manipulation doesn't.

The Problem

Reading traffic: Not the issue.

Manipulating it reliably: Not feasible.

This is an industry-wide problem, not unique to us.

Speed and accuracy matter, but they matter less than the fact that the core interception path is just too fragile.

The Pivot

Extension is the only reliable way to modify webpage content on AI sites. Desktop engine still matters — just differently.

3/5

The Real Opportunity

Hybrid architecture + enterprise pull

Mechanism Network interception → Extension + native messaging
Desktop role Interception → Local engine + model
Framework Tauri → Electron (reliability)

Vistage Enterprise Signal

Internal champion recommended Sonomos. They have shadow AI concerns, no incident yet. Large company, warm intro, active problem.

Chrome native messaging: ✓ Working — extension talks to desktop engine.
