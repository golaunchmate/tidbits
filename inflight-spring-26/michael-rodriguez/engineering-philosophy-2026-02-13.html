2/5
Debugging Philosophy

Manual first, AI as backup

Don't give all control to AI. Your brain is the primary debugger.

The Process

  • Reproduce the bug by manipulating database state
  • Read error logs manually — follow the line numbers
  • Test APIs directly with curl to isolate the problem
  • Use Supabase to inspect live database state
  • Ask AI when you're stuck, but review suggestions critically

When AI gives conflicting info, test the actual API directly to confirm what works. The AI might be wrong about implementation details — your curl test won't lie.

3/5
Development Cycle

User story → Objective → Tasks

Framework

User Story: "As a [type of user], I want to [perform task] so I can [achieve goal]"

Technical Objective: What does the system need to do to enable this?

Developer Tasks: Break it into concrete implementation steps

Features often reveal themselves through bug discovery. A user hitting an error state? That's not just a bug — it's a missing feature waiting to be built.

4/5
Testing Strategy

Incremental and reversible

Don't merge big changes without verifying they work. Don't undo everything immediately when something breaks.

Key Tactics

  • Cursor checkpoints: Click the reverse button on any message to undo all changes after that point
  • Database manipulation: Set users to specific states (delete agents, toggle flags) to simulate edge cases
  • Test in isolation: Use curl to test APIs independently from the full app
  • Go to definition: Right-click functions to jump to their source — trace the logic manually

Balance "vibe coding" (accepting all AI changes) with careful review. Know when to lean on each.

5/5
Architecture Mental Model

How the pieces actually connect

The Flow

1.Client-side: React/Next.js in the browser (button clicks, UI)
2.API routes: Intermediaries that protect sensitive data
3.Services layer: Core business logic on the server
4.Database/External APIs: Supabase, Letta, etc.

Key insight: Services contain the logic. API routes are just gatekeepers. If you're debugging, start at the service layer — that's where the real work happens.
