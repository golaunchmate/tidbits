WEEK 2-3 · Jan 14-27

From skills to infrastructure

Started building the tidbits pipeline. This is where it got interesting.

I have a shared tidbits agent that creates weekly narratives by gathering context from each founder's agent. Multi-step back-and-forth conversations.

The Problem

Multiple founders triggering tidbits concurrently could cause flow interleaving. Letta has a conversations API — I figured: one conversation per founder on the tidbits agent. Parallelism solved?

Not quite.

5/6
WEEK 3 · The Discovery

A2A response routing is broken

Native agent-to-agent messaging routes to an agent, not a conversation.

What Actually Happens

✓ Tidbits sends to founder agent — works fine

✓ Founder agent processes and responds

✗ Response uses default a2a → goes to tidbits' main conversation

The sub-conversations exist. But founder responses don't route back to them. All responses pile into the default conversation.

Solution: Custom send_message_to_agent_conversation tool that founder agents use to route responses to their specific sub-conversation.
