The Architecture

Capabilities via Letta Conversations

The insight: separate capability management from main chat.

Sidebar — decodes Letta core block, shows current capabilities
Capabilities conversation — dedicated thread for adding/editing skills
Main conversation — stays clean, focused on founder work
Activation — when triggered, capability runs in main chat

User gets visibility + control. Chat history stays uncluttered.

5/5
The Next Level

LLM-generated activation

Instead of canned "activate [skill]" prompts, the capabilities agent crafts contextual activation messages via tool call.

Flow

1. User clicks capability in sidebar

2. Hits capabilities conversation

3. LLM crafts activation prompt

4. Tool call drops it into main conversation

5. Main agent runs the skill

The activation is intelligent, not static. The capabilities agent becomes the skill launcher.

Building: send_to_main_conversation(message) — the inverse of a2a routing.
