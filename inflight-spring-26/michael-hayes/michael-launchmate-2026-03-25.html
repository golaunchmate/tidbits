Effort Visibility in Autonomous Agents

Michael Hayes · LaunchMate

Building scheduled tasks that run autonomously led to a realization: good output isn't enough if founders can't see the work that produced it.

scroll to continue ↓
1/5

The Starting Point

I thought I was building scheduled reminders — simple prompts that fire at specific times to nudge founders into action.

But as I wrote the prompt spec, I kept thinking about the autonomous execution patterns from Andrej's podcast: agents running experiments overnight, auto-research loops, Peter Levels' 10 parallel Codex agents doing real work without supervision.

The question shifted: What if these aren't reminders at all? What if they're full autonomous executions — founders wake up to completed work, not just notifications?

2/5

The ChatGPT Problem

Here's the tension: ChatGPT produces incredibly good output, but founders still can't attribute value to it.

Craig's friction analysis (F7+F14) identified this as the "meaning" problem — even when AI helps, founders don't perceive the value because they don't see the effort.

When I manually generate a tidbit, Michael sees:

  • Me scrolling through 42 messages
  • Highlighting key decisions
  • Writing multiple drafts
  • Tweaking tone and structure
  • Publishing and updating memory

That's 2 hours of visible labor. When an agent just returns a URL, that perception disappears.

3/5

The Decision: Surface the Work

Autonomous execution needs execution transparency — not just the deliverable, but a log of every step completed.

I added this to the poke schema:

✓ Searched last 7 days (42 messages)
✓ Identified key decision: MCP architecture
✓ Extracted trade-off: centralized vs isolated tidbits
✓ Generated 4-slide tidbit (850 words, engineer tone)
✓ Published to GitHub Pages
✓ Updated interaction_history memory

Here's your tidbit: [URL]

Now founders see 6 checkmarks of labor before the result. The "wow this is incredible" moment comes from perceiving effort, not just quality.

4/5

Where Things Stand

Shipped: Agent Pokes with autonomous execution specs — founders can schedule multi-step workflows (tidbit generation, mentor syncs, research) that run without human intervention.

Key addition: execution_plan.steps array + sample_output in schema. Execution-side prompt must log each step before surfacing deliverable.

Open question: Tidbits storage architecture. Currently nested HTML files (good for GitHub Pages, human browsing). But if tidbits are "historical magic" that multiple agents query for coordination, should they move to Supabase?

Trade-off: simplicity vs cross-agent query performance. Deferred until we validate whether coordination actually requires cross-founder thematic queries like "show all market validation decisions across cohort."

Next: Build execution-side prompt that receives poke spec and runs autonomous workflow with full logging.
