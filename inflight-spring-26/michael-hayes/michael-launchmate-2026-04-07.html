The Connector Pivot

When your product isn't the interface — it's the memory behind every interface.

A team meeting where we asked the question we'd been avoiding: what happens when founders don't need to type anymore?

1/5
scroll ↓
The problem we'd been avoiding

Founders graduate

Three AI revolutions hit while we were building LaunchMate:

RevolutionWhat it proved
ChatGPTAI can be super intelligent
Claude CodeAI can code — generative, project-based
OpenClaudeGeneralized agent with compounding context

Each one made LaunchMate's chat interface less special. Founders discover Claude Code, start building with it, and suddenly they're not typing updates into our app anymore.

We lose visibility. Their agent loses context. And we can't blame them — why would they manually update us when their tools already know what they did?

"The one thing through everything we've done is it's required the human to type the update. If we get around that using connectors, we could see really cool benefit."
2/5
The insight that reframed everything

Memory is the moat

Craig made an observation that stuck: two years into ChatGPT, and we're still coordinating via email, video calls, and text messages. AI made individuals smarter, but teams still sync the old way.

The multiplayer problem is unsolved. And that's where LaunchMate actually lives.

The Letta argument

Harnesses and LLMs are a commodity. Memory is what you should own.

This hit hard. We'd been optimizing the chat interface — making it snappier, adding Discord, polishing the experience. But the chat isn't the value. The context that lives behind it is.

What if founders never visited LaunchMate at all? What if they just connected their tools, kept working where they already work, and advisors got everything they needed from the dashboard?

3/5
The pivot

From chat app to memory hub

Current modelProposed model
Founders type updates into LaunchMateConnectors auto-populate from existing tools
Chat interface is primaryDashboard is primary (for advisors)
Lose founders when they graduateStay connected via MCP hooks

The technical path: MCP server. Once that exists, any agent — Claude Code, Cursor, Poke, Letabot — can inject context into LaunchMate agents. Neal wants to dump transcripts via Poke. Same pattern.

And for the proactive side: Pokes + Discord notifications become the outbound channel. Daily briefings, weekly assessments, prompts to try in other tools. Ship the kitchen sink, see what sticks.

"I could see there being no chat interface for LaunchMate for the founder. It's purely: what tools do you use? Connect, connect, connect. Start working."

Where we landed

Unresolved (and that's okay)

Three tensions we're sitting with:

  • Chat vs. Dashboard: Should founders ever visit LaunchMate web, or just let it run in the background?
  • Letta Code integration: If Letta goes all-in on local agents, how does that benefit our cloud?
  • Memory ownership: If memory is the moat, how do we become the hub across ALL tools?
